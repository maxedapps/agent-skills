# Complexity gates

Candidate signals only — admit via the full admission gate in `SKILL.md`. Prefer `delete → direct use → local → existing shared → new abstraction`. Uncertain need/reachability/behavior → Validate or Ask user, don’t assert removal.

| Surface | Candidate signals | Justified exceptions |
|---|---|---|
| **Source / local structure** | Pass-through helpers; trivial wrappers; rename-only type guards; tiny nav-only files; speculative factories/adapters/options/generics/hooks; condition-laden shared abstractions; duplicate state/ownership | Intention-revealing domain language; useful narrowing; stable policy; real module/trust boundaries; external contracts; explicit lifecycle/cleanup ownership; established (not incidental) duplication |
| **Plans / proposals** | Hypothetical users/scale; unrequested flexibility; premature migration/rollout/plugin/fallback/generalization; safeguards without reachable failure; validation heavier than the decision | Current acceptance; known consumers; approved migration/rollout; destructive/irreversible effects; measured constraints; explicit user/authority decisions |
| **Architecture / data flow** | Forward-only layers; parallel models/duplicate SoT; adapters around one impl; broad registries/locators; parameters/branches encoding unrelated cases | Stable domain boundaries; centralized policy; external interfaces; real isolation/ownership; structured lifecycle; established shared behavior unlikely to diverge |
| **Review findings / fixes** | Broaden scope; add frameworks/layers; harden unreachable paths; generalize a local fix; checks/state without failure path + owner | Evidenced defects; required authority coverage; trust-boundary/data invariants; measured risks; destructive/non-idempotent sides. Preserve finding IDs; owner decides |
| **Tests / validation** | Broad DSLs; deep fixtures; excessive mocks; combinatorial matrices without distinct behavior; duplicated asserts; helpers hiding setup/outcomes; speculative config combos | Unique protection for behavior, wiring, parsers, compat, security/data, concurrency, cleanup, upgrade canaries, irreversible effects. Repetition can beat clever test abstraction |
| **Configuration** | Unused flags/modes/env/overrides; invalid combos; single-value knobs; config duplicating source state | External deploy contracts; distinct supported envs; safe ops controls; secret boundaries; named owners + exercised combos |
| **Dependencies / tech** | Novelty for trivial local behavior; overlapping packages; new infra before stack insufficient; wrappers without boundary | Demonstrated capability/security/compat/maintenance/ownership advantage after replacement cost, ecosystem, license, ops support |
| **Concurrency / scale** | Unmeasured queues/caches/retries/locks/workers/batching/sharding/backpressure; coordination for hypothetical traffic; state machines for serial work | Measured constraints or realistic overlap. Need: realistic overlap + shared mutable state or non-idempotent side effect + meaningful harm. User count alone proves nothing. Keep structured concurrency/cancel/cleanup/reconciliation/idempotency when model requires |
| **Validation / security** | Duplicate caller+callee checks; ceremonial validation after trusted parse; scattered authz; defensive branches with no reachable input | Real trust-boundary validation; authn/authz/data invariants/safe parsing per threat model. Prefer carrying validated knowledge over deleting necessary controls |
| **Fallbacks / compatibility** | Fallback chains; silent alternates; unsupported legacy; adapters without consumer/contract/removal condition | Real compat commitments; external contracts; staged migrations; recovery; fallbacks tied to observed failures |
| **Operations / resilience** | Unmeasured retries/caches/queues/breakers/flags/dashboards/logs/recovery/deploy layers; duplicated observability; safeguards without operator action | Measured limits; required availability; useful operator decisions; destructive/irreversible effects; unknown outcomes; reconciliation/rollback/recovery with clear ownership |
